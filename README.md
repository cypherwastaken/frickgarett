# FrickGarett 
Click [here](./DOCUMENTATION.md) to view the docs

| Feature / Aspect                          | Version 1                                        | Version 2 (Improved)                                                                                   | Why Version 2 is Better                                                                                       |
|-------------------------------------------|-------------------------------------------------|--------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------|
| **CDN Subdomain Rewriting**                | ❌ Not handled                                   | ✅ Rewrites `*.tiktokcdn.com` / `*.tiktokcdn-us.com` → `www.tiktok.com`                                  | Prevents 404 errors when TikTok serves JS/CSS/video from their CDN subdomains.                                 |
| **Module Loader / Dynamic Imports**       | ❌ Not handled                                   | ✅ Supports module rewriting through `rewriteJS`                                                        | Ensures dynamically loaded JS modules work through the proxy without `SyntaxError`.                            |
| **Service Worker Blocking**               | ❌ Not blocked                                   | ✅ Calls `navigator.serviceWorker.getRegistrations().then(...unregister())`                               | Stops service workers from interfering with the proxy or caching wrong resources.                              |
| **Extended HTML Rewriting**               | ✅ Basic rewriting (`src`, `href`, `data-src`)  | ✅ More extensive: also rewrites `data-href`, `data-url`, `poster`, `action`, `srcset`, `form`, `meta refresh`, `window.location`, `fetch`, `WebSocket`, `url()` | Covers almost all URL references in HTML to ensure everything goes through the proxy.                         |
| **Fetch / XHR Proxying**                  | ✅ Basic override                                | ✅ Fully overrides `fetch`, `XMLHttpRequest`, `history.pushState`, `history.replaceState`, `window.location.assign` | Makes JS-based navigation and API requests go through the proxy, improving functionality for single-page apps. |
| **Timeout / AbortController**             | ❌ Not implemented                               | ✅ Uses `AbortController` with 20s timeout                                                             | Prevents hanging requests and improves stability on slow endpoints.                                            |
| **Headers / Cookie Forwarding**           | ✅ Basic headers (`user-agent`, `accept`, `accept-language`, `referer`) | ✅ Extended headers (`sec-fetch-*`, `upgrade-insecure-requests`) + forwards cookies                      | Mimics a real browser more accurately, reduces blocking or misbehaving responses.                              |
| **Cache Control**                          | ❌ Not set                                      | ✅ Sets `cache-control: public, max-age=3600`                                                           | Enables client caching for repeated requests, improving speed.                                                 |
| **Blocked Headers Removal**               | ✅ Removes `content-security-policy`, `x-frame-options`, etc. | ✅ Same as v1                                                                                           | Maintains compatibility with CSP and framing restrictions.                                                     |
| **JavaScript Rewriting**                  | ❌ No `rewriteJS` integration                    | ✅ Integrates `rewriteJS` for inline and external JS                                                     | Ensures dynamically executed code is also rewritten for proxy compatibility.                                   |
| **CSS URL Rewriting**                     | ✅ Basic `url()` rewriting                        | ✅ Same, but integrated with CDN subdomain rewriting                                                   | Prevents broken assets (images/fonts) loaded via CSS.                                                         |
| **Error Handling / Logging**              | ✅ Basic, returns 500                             | ✅ Same, keeps try/catch, includes `console.error`                                                     | Easier debugging during proxy failures.                                                                       |
| **Overall Reliability on TikTok / JS-heavy sites** | ⚠ Partial, often fails on TikTok Web            | ✅ Strong, handles CDN, service workers, dynamic modules, XHR, history API                              | Version 2 works with modern JS-heavy websites, including TikTok Web, without common 404 / syntax issues.      |
